# ArcLink Remote - 局域网手动测试指南 (Manual Test Plan)

本文档旨在指导测试人员如何在同一局域网（LAN）内的两台 Windows 10/11 电脑上部署并运行 ArcLink Remote MVP 进行真实的远程桌面控制测试。

---

## 1. 测试前准备工作 (Pre-requisites)

### 1.1 硬件要求
* **电脑 A (被控端 - Host)**: 运行 Windows 10 或 Windows 11 的物理机或虚拟机。
* **电脑 B (控制端 - Viewer)**: 运行 Windows 10 或 Windows 11 的物理机或虚拟机。
* **局域网网络**: 两台电脑必须连接到同一个路由器/交换机（无线 Wi-Fi 或有线以太网），确保彼此可以 ping 通。

### 1.2 系统网络配置（关闭防火墙或放行端口）
由于 ArcLink 使用自定义 TCP/UDP 端口进行握手与流传输，请在 **电脑 A (Host)** 上的 Windows Defender 防火墙中放行以下端口，或者在测试期间临时关闭防火墙：
* **TCP 8443** (控制握手端口)
* **UDP 8444** (视频流媒体传输端口)

---

## 2. 编译并部署二进制文件 (Build & Deploy)

### 2.1 本地编译
在您的开发环境中，运行以下命令编译 Release 版本的二进制可执行文件：
```bash
# 导航到 Rust Workspace 目录
cd arclink-remote

# 编译 Host 与 Viewer
cargo build --release
```

### 2.2 复制二进制包到测试机
编译完成后，提取以下路径的二进制可执行文件：
* **Host 被控端**: `arclink-remote/target/release/arclink-host.exe`
* **Viewer 控制端**: `arclink-remote/target/release/arclink-viewer.exe`

将 `arclink-host.exe` 复制到 **电脑 A** 的桌面上。  
将 `arclink-viewer.exe` 复制到 **电脑 B** 的桌面上。

---

## 3. 手动测试步骤 (Execution Steps)

### 步骤 3.1: 启动被控端服务 (电脑 A - Host)
1. 在 **电脑 A** 上双击运行 `arclink-host.exe`。
2. 程序启动后，将展现明亮优雅的 **Apple Liquid Glass** 主控制界面。
3. 在界面中，观察并记录 **局域网 IPv4 地址** (例如：`192.168.1.105`) 与 **控制监听端口** (默认：`8443`)。
4. 点击界面下方的 **[ 开启局域网监听 ]** 按钮。
5. 此时，右上角的状态胶囊应立即由 **未启动 (Offline)** 变更为 **准备就绪 (Listening)**，代表 TCP 服务已在后台安全等待。

### 步骤 3.2: 启动控制端连接 (电脑 B - Viewer)
1. 在 **电脑 B** 上双击运行 `arclink-viewer.exe`。
2. 程序启动后，进入 Viewer 首页面。
3. 在 **被控端 IP** 输入框中，填入刚才记录的电脑 A 的局域网 IP (例如 `192.168.1.105`)。
4. 在 **控制监听端口** 输入框中，填入 `8443`。
5. 点击下方大号的 **[ 开启远程控制会话 ]** 主按钮。
6. 右上角状态胶囊变更为 **正在握手直连...**。

### 步骤 3.3: 确认安全接入控制授权 (电脑 A - Host)
1. 观察 **电脑 A (Host)**，其界面中央应弹出一个高亮毛玻璃效果的 **局域网接入申请** 确认窗口。
2. 窗口中会准确列出申请者的 **主控端 ID** 与其所在的 **局域网 IP**。
3. 点击弹窗中的 **[ 接受连接 ]** 按钮。
4. 此时，Host 界面右上角状态胶囊变更为 **远程主控连接中 (Active)**，并在界面上实时展现当前网络延迟、采集率、会话时长等状态卡片。

### 步骤 3.4: 观察远程画面并进行交互 (电脑 B - Viewer)
1. 观察 **电脑 B (Viewer)**，其页面应立即载入 **电脑 A (Host)** 的真实 Windows 桌面画面。
2. 在 Viewer 的控制区域内移动鼠标、进行左键或右键点击、滑动鼠标滚轮，观察电脑 A 上的鼠标光标是否发生相同的物理偏移与点击操作。
3. 激活 Viewer 窗口，按下键盘按键（例如 `Space`、`Enter` 等），观察电脑 A 上的文本输入区是否响应键盘映射。
4. 移动鼠标到 Viewer 窗口上方，悬浮触发 **控制悬浮条**。
5. 勾选 **实时性能指标** 选框，检查 RTT 延迟数值与视频解码帧率（FPS）是否来自真实的实时计算，而不是伪造数据。

### 步骤 3.5: 断开远程控制会话 (Session Termination)
我们可以采用以下三种方式之一测试断开连接的清理机制：
1. **主控端主动断开**: 在电脑 B (Viewer) 的顶部悬浮控制条中，点击 **[ 结束控制 ]**。
2. **被控端强行中断**: 在电脑 A (Host) 的被控状态卡片中，点击 **[ 立即断开控制会话 ]**。
3. **拔掉网线/网络异常**: 模拟网络中断 5 秒，观察两端的心跳超时检测（5 秒阈值）是否能自动触发会话清理并安全归位到 Listening 状态。

## 4. 故障排除指南 (Troubleshooting)

| 异常现象 | 可能的原因 | 排查与解决方法 |
| :--- | :--- | :--- |
| **Viewer 提示 Connection Reset / Error** | 1. 被控端未开启监听<br>2. 局域网 IP 填写错误<br>3. Windows 防火墙拦截 | 1. 检查 Host 状态胶囊是否显示为 Listening<br>2. 重新核对 Host IP 地址<br>3. 关闭 Windows 防火墙或添加放行规则 |
| **画面能显示，但鼠标点击无效** | 1. 被控端未使用管理员权限运行<br>2. 未点击接受授权 | 1. 部分受 UAC 保护的程序需要 Host 以 **管理员身份运行** 才能注入模拟输入<br>2. 确保在 Host 弹窗中点击了“接受” |
| **画面一直卡在 Loading 状态** | 1. UDP 8444 端口被占用或被防火墙拦截 | 1. 确保放行了 **UDP 8444** 端口<br>2. 检查日志面板是否打印了 “UDP Bind Error” |
| **程序闪退或 panic** | 1. 系统没有兼容的 D3D11/GDI 资源 | 1. 本 MVP 已提供高度可靠的 GDI 兼容层作为默认捕获，可在任何 Windows 环境下安全运行 |
